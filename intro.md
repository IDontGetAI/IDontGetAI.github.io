# IDontGetAI 项目介绍

> **"I Don't Get AI"** - 在认知的迷雾中探索，在技术的故障中思考

---

## 🧠 核心理念与目标

### "I Don't Get AI" 的探索精神

**IDontGetAI** 不仅是一个个人知识库，更是一种对人工智能时代的诚实态度。在 AI 技术飞速发展的当下，我们承认**理解的不完整性**，拥抱**认知的不确定性**。这个网站记录了跨学科探索的轨迹——从数学分析到哲学思辨，从计算机科学到认知心理学，每一个主题都是对"我还不懂"的回应。

### "Cognitive Glitch" 设计哲学

**Cognitive Glitch（认知故障）** 是本站的核心美学主题：
- **视觉层面**：采用赛博朋克风格的故障艺术（Glitch Art），通过 RGB 色彩分离、扫描线、HUD 框架等元素，营造出"系统正在解码"的氛围
- **认知层面**：象征人类在理解复杂系统时的断裂感，知识不是完美的拼图，而是充满裂隙的网络
- **哲学层面**：故障不是缺陷，而是探索的证据——正是这些"glitch"让我们意识到思维的边界

## ⚡ 功能特性

### 1. 智能文档查看系统

#### Markdown 阅读器
- **远程加载**：支持从 GitHub 仓库动态加载 `.md` 文件，实现内容与代码分离
- **富文本渲染**：基于 `markdown-it` 引擎，支持 LaTeX 数学公式、代码高亮（`highlight.js`）、表格等
- **图片处理**：自动将相对路径转换为 GitHub 完整 URL，确保图片正常显示
- **一键下载**：下载功能将图片链接转换为绝对路径，离线阅读无障碍

#### PDF 阅读器
- **原生渲染**：使用 `pdfjs-dist` 实现浏览器原生 PDF 渲染，无需第三方服务
- **混合加载引擎**：
  - 优先使用 Blob API 获取 GitHub 文件并渲染
  - 降级策略：直接使用 iframe 加载原始 URL
- **沉浸式体验**：深色玻璃拟态 UI，与网站主题无缝融合
- **全功能支持**：保留浏览器原生的搜索、打印、缩放功能
- **全局路由**：`/pdf-viewer?url=[encoded_url]` 支持外部调用

### 2. 无头 CMS 内容管理

- **GitHub 同步**：内容托管在独立的 GitHub 仓库，修改后自动同步，无需重新构建网站
- **数据驱动设计**：前端只负责展示，内容管理完全解耦
- **填空式教程**：
  - `add_github_md.md`：指导如何添加远程 Markdown 笔记
  - `add_pdf.md`：指导如何集成 PDF 资源
  - 降低内容贡献门槛，专注于知识创作

### 3. 下载与离线功能

- **源文件下载**：
  - Markdown 文件：处理后的内容保留完整格式和图片链接
  - PDF 文件：直接下载原始文档
- **智能文件名**：使用页面标题（title）作为文件名，降级到 URL 提取
- **一致 UI**：下载按钮统一布局（左侧内容操作区 + 右侧辅助功能区）

## 🛠️ 技术架构

### 核心技术栈

#### 前端框架
- **React 19**：最新特性，支持 Compiler 和 Server Components（未来扩展）
- **TypeScript 5.9**：类型安全，智能提示
- **Vite 7**：极速开发体验，HMR（热模块替换）

#### UI 系统
- **Tailwind CSS v4**：原子化 CSS，自定义主题配置
- **shadcn/ui**：基于 Radix UI 的组件库，完全可定制
- **Framer Motion**：流畅的动画系统

#### 路由与状态
- **wouter**：轻量级 Hash 路由（兼容 GitHub Pages 静态托管）
- **React Context API**：全局主题管理（`ThemeContext`）

#### 文档处理
- **markdown-it**：Markdown → HTML 渲染引擎
- **highlight.js**：代码语法高亮
- **pdfjs-dist**：Mozilla PDF.js，浏览器原生 PDF 渲染
- **KaTeX** (间接支持)：LaTeX 数学公式渲染

#### 表单与验证
- **react-hook-form**：高性能表单管理
- **zod**：TypeScript 优先的模式验证

### 项目结构

```
src/
├── components/          # React 组件
│   ├── ui/             # shadcn/ui 组件库
│   ├── RemoteNoteLayout.tsx    # Markdown 阅读器
│   ├── PdfViewer.tsx           # PDF 阅读器
│   ├── TerminalNavbar.tsx      # 终端风格导航栏
│   └── ...
├── pages/              # 页面组件
│   ├── Math.tsx        # 数学学科页面
│   ├── AI.tsx          # AI 学科页面
│   └── notes/          # 笔记详情页
├── contexts/           # React Context
│   └── ThemeContext.tsx
├── lib/                # 工具函数
│   └── utils.ts
└── styles/             # 全局样式
    └── cogito.css      # Cognitive Glitch 主题
```

### 构建与部署

- **开发命令**：`npm run dev` → 本地开发服务器（默认 `localhost:5173`）
- **构建命令**：`npm run build` → 生成静态文件到 `dist/`
- **部署方案**：`gh-pages` 自动部署到 GitHub Pages
- **路径别名**：`@/` 映射到 `src/`，简化导入路径

## 💾 内容管理系统

### 无头 CMS 工作流

1. **内容创作**：
   - 在独立的 GitHub 仓库中编写 Markdown/上传 PDF
   - 使用标准 Markdown 语法，图片使用相对路径

2. **内容集成**：
   - 在网站代码中配置资源 URL（GitHub raw 链接）
   - 更新路由映射（如 `/notes/math/analysis`）

3. **动态加载**：
   - 用户访问时，前端从 GitHub API 拉取最新内容
   - 实时渲染，无需重新构建网站

4. **版本控制**：
   - 内容仓库的 Git 历史记录所有修改
   - 可回溯到任意历史版本

### 填空式教程机制

- **add_github_md.md**：
  - 复制模板 → 填写标题/URL → 更新路由配置
  - 示例：添加"数学分析笔记"只需 3 步

- **add_pdf.md**：
  - 上传 PDF 到 GitHub → 获取 raw 链接 → 配置 PdfViewer 组件
  - 支持大文件（GitHub LFS）

## 🎨 设计风格

### 视觉语言

#### 配色方案
- **主色调**：霓虹绿 `#00ff80` - 象征代码、数据流、神经网络
- **背景色**：深黑渐变 `#0a0a0f` → `#050508` - 赛博朋克的深邃虚空
- **辅助色**：
  - 青色 `#00d3ff` - RGB Glitch 效果（Cyan 通道）
  - 洋红 `#ff2bd6` - RGB Glitch 效果（Magenta 通道）

#### Glitch 美学元素
- **RGB 色彩分离**：核心元素（如 favicon）使用 3 层偏移，营造"信号干扰"效果
- **扫描线**：CRT 显示器风格的水平线（opacity: 0.08-0.15）
- **HUD 框架**：四角 L 型线条，模拟科幻界面
- **玻璃拟态**：半透明背景 + 毛玻璃模糊（`backdrop-blur`）

#### Favicon 设计
- **核心概念**：神经网络节点集群
  - 中心节点（5px）+ 5 个卫星节点（2.2-3.2px）
  - 实线连接表示神经通路
  - 虚线连接表示"认知断裂"
- **设计寓意**：
  - 去中心化的 AI 认知
  - 不对称结构象征探索的混乱性
  - 无问号，纯粹的技术符号

### 设计原则

1. **极简主义**：去除冗余装饰，聚焦内容本身
2. **科技感**：终端式导航栏、等宽字体、命令行美学
3. **暗色主题**：保护眼睛，突出霓虹色彩
4. **响应式设计**：移动端自适应，平板/桌面优化布局
